Git是往后工作、生活、学习和交流中都会经常用到的远程仓库、版本管理系统。功能十分之强大，所以在这里稍微做个总结。

  2. 安装完了之后注册并登陆github，初始化git配置，参考资料:
    move to

  3. 看了上一篇参考资料，基本git基础的操作就已经差不多掌握了。而在具体使用中会遇到的一些问题，我再稍作总结。

    1. 经常遇到push不上去的情况，很多时候是远程仓库与本地仓库不同步导致的，这时候使用git pull把远端仓库复制一份到本地，再进行push就好。（pull仅会复制本地没有改动过的文件到本地，所以不必担心改动被覆盖）
    2. 使用commit时，如果没有加 –m “。。。”会进入一个编译工具里面，其实这个工具挺强大的，不过新人看到一般都会慌，不知道咋退咋改。其实按一下i就可以进入编辑状态，而你输入的内容则会作为git commit的版本信息，方便版本查找。在编辑状态下按esc退出编辑状态，然后按“：”进入命令模式。我常用的命令只有两个，w是保存，q是退出。如“：wq”意味着保存并退出。
    3. Vi可以进入文件编辑工具，在里面写代码、改代码都很简单。退出方式跟上面一样。
    4. Cd是去到某个文件夹。
    5. 因为好奇，实验过回溯版本，但结果告诉我，若非情不得已，切莫回溯版本。因为几个月前用过一次，之后就决定再也不用，现在也已经忘差不多了，所以回溯的方法咱也就不多说了。如果实在想知道——Git入门之回溯版本move to
